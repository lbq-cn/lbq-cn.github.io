# sidebar 侧边栏配置

所有配置项写入config.js生效。

# 默认配置

config.js中压根不写sidebar配置项，则根据页面标题自动生成目录，一级标题为根，二级标题、三级标题为枝叶。

如果要写该配置项，参数值为auto，等同于默认配置。

sidebar:'auto'

缺点是只有一个一级菜单。

# 手动配置

侧边栏参数值是一个数组[]，元素是对象{}，每个对象是一个独立的侧边栏菜单sidebarItem。

每个侧边栏菜单有一个标题参数项text，一个链接参数项link，一个子菜单参数项children，一个是否可折叠参数项collapsible

​ - text的值将替代link指向页面的一级标题

​ - link的值为具体的页面，以/结尾将代表该路径下的readme.md，后缀.md可省略

​ - children参数项的值，同sidebarItem

​ - collapsible配置为true后，可将子菜单children折叠

sidebar:[{
      text:'个人笔记站点制作',
      collapsible:true,
      children:
      [
        {
          text:'站点开发',link:'/'
        },
        // 侧边栏配置也可以直接写字符串，link 指向字符串，text 为页面的一级标题。
        '/部署.md'
      ]
    },
    {
      text:'vuepress配置',
      collapsible:true,
      children:[
      {
        text:'侧边栏配置',
        link:'/vuepress_sidebar',
      }]
    }],
